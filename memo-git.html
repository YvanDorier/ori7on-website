Mémo Git & GitHub

Les 5 concepts essentiels

Les concepts

Commit

Sauvegarder une version

Tu modifies des fichiers, puis tu fais un commit pour dire : "Je sauvegarde ces changements avec un message explicatif."

Comme appuyer sur "Enregistrer" avec un commentaire dans Word.

Branch

Une copie de travail isolée

Une branche est une copie parallèle du code. On y travaille sans toucher à main, pour ne pas casser ce qui fonctionne.

Comme faire une photocopie d'un document avant de le corriger au stylo rouge.

Push

Envoyer vers GitHub

Push = envoyer tes commits depuis ta machine vers GitHub. Sans push, tes modifications restent invisibles en ligne.

Comme envoyer un e-mail : le brouillon reste local jusqu'à ce que tu appuies sur "Envoyer".

Pull Request

Demander à fusionner

Une PR est une demande formelle sur GitHub pour intégrer ta branche dans main. C'est là qu'on relit, commente, approuve.

Comme soumettre un rapport à ton responsable avant qu'il soit officiel.

Merge

Fusionner dans main

Merge = valider la PR et intégrer définitivement les changements dans main. C'est l'étape finale.

Ton responsable signe le rapport et il devient officiel.

Le flux complet

1
Modifier les fichiers du projet
2
Commit — sauvegarder avec un message
3
Push — envoyer la branche sur GitHub
4
Pull Request — ouvrir une demande de fusion
5
Relecture — révision et approbation
6
Merge — fusionner dans main ✓
